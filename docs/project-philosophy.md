# Milan Red Line - Project Philosophy

## Our Goal

Build a premium website for short-term rentals that is elegant, fast and easy to maintain.

The project is designed to become a reusable platform for future properties.

---

# Core Principles

## Simplicity

Choose the simplest solution that solves the problem well.

Avoid unnecessary complexity.

---

## Data before Code

The website should display data, not contain data.

Apartment information belongs in the database.

---

## User Experience

Every design decision should improve the guest experience.

The website should answer questions before the guest asks them.

---

## Privacy

The exact apartment address is NOT displayed publicly.

Only the general area, nearby metro station and approximate location are shown.

The complete address is communicated only after booking.

---

## Transparency

Legal information must always be available.

Each apartment page will clearly display:

- CIN

- CIR (when applicable)

---

## Quality over Quantity

A few excellent pages are better than many average ones.

Every photo, every text and every section should have a purpose.

---

## Reusability

The platform should allow adding a new apartment without writing code.

The ideal workflow is:

1. Create apartment

2. Fill information

3. Upload photos

4. Publish

---

## Documentation

Important decisions are documented.

The project should be understandable even after several months.

---

## Long-Term Vision

Milan Red Line is the first project.

The architecture should be reusable for future websites managing apartments, holiday homes or B&Bs.
