# # Milan Red Line — Future Features

*Last updated: July 2026*

---

# Purpose

This document collects ideas that are intentionally **outside the current roadmap**.

These features are not priorities today.

They should only be considered once the website has established solid organic traffic, regular direct bookings and a stable technical foundation.

---

# Guest Experience

Improve the stay before, during and after arrival.

Possible features:

- Digital house manual

- Online check-in

- Self check-in instructions

- Local recommendations

- Emergency information

- Interactive apartment guide

- Guest area

---

# Property Management

Reduce manual work as the project grows.

Possible features:

- Apartment database (Supabase)

- Image management

- Dynamic amenities

- Occupancy calendar

- Reservation overview

- Internal notes

- Document management

---

# Content Platform

Expand the editorial experience.

Possible features:

- Guide search

- Topic filters

- Saved guides

- Suggested itineraries

- Personal travel planner

- AI-assisted multilingual workflow

---

# Marketing

Strengthen long-term relationships with guests.

Possible features:

- Newsletter

- Returning guest offers

- Seasonal travel updates

- Local event recommendations

- Repeat guest programme

Always privacy-first.

Only with explicit consent.

---

# Analytics

Understand visitor behaviour.

Possible future improvements:

- Scroll depth analysis

- Search behaviour

- Guide conversion paths

- Returning visitors

- A/B testing

- Funnel analysis

Decisions should always be data-driven.

---

# Visual Identity

Continue improving the brand experience.

Ideas:

- Custom illustrated map

- Destination illustrations

- Better apartment photography

- Small motion effects

- Icon library

- Branded illustrations

Only if they improve clarity.

Never decoration for its own sake.

---

# Scalability

The platform should continue working even if:

- more apartments are added;

- more languages are introduced;

- hundreds of guides are published;

- traffic increases significantly.

Growth should require configuration rather than architectural changes.

---

# Decision Rule

Before implementing any feature, ask:

- Does it improve the guest experience?

- Does it strengthen trust?

- Does it increase direct bookings?

- Can it be maintained easily?

- Is it more valuable than publishing another high-quality guide?

If not, postpone it.

---

# Philosophy

Features do not create value on their own.

Useful content, trust and a great guest experience do.

Every future improvement should support those three principles.
